# mTLS Authentication Specification

## Overview

This document specifies the mutual TLS (mTLS) authentication system for the Edge-Cloud Orchestrator, ensuring secure, bidirectional authentication between the Control Plane and Edge Nodes.

## Authentication Flows

### Flow 1: Node Registration (Initial Trust Establishment)

```
┌─────────────┐                                    ┌─────────────┐
│  Edge Node  │                                    │ Control Plane│
│   (Agent)   │                                    │   (CA)       │
└──────┬──────┘                                    └──────┬──────┘
       │                                                  │
       │ 1. Generate key pair + CSR                       │
       │    (node-id, region, hardware-info)              │
       │                                                  │
       │ 2. POST /api/nodes/register                      │
       │    { csr, metadata, bootstrapToken }             │
       │─────────────────────────────────────────────────>│
       │                                                  │
       │ 3. Verify bootstrap token                        │
       │    (one-time use, expires in 1 hour)             │
       │                                                  │
       │ 4. Sign CSR → Node Certificate                   │
       │    (validity: 90 days)                           │
       │                                                  │
       │ 5. Return { certificate, caCert, config }        │
       │<─────────────────────────────────────────────────│
       │                                                  │
       │ 6. Store certificate securely                    │
       │    (encrypted at rest)                           │
       │                                                  │
```

**Bootstrap Token Security:**
- Generated by Control Plane admin
- One-time use only
- Expires in 1 hour
- Delivered via secure channel (e.g., cloud provider metadata service)

### Flow 2: Mutual Authentication (Every Connection)

```
┌─────────────┐                                    ┌─────────────┐
│  Edge Node  │                                    │ Control Plane│
│   (Agent)   │                                    │   (API)      │
└──────┬──────┘                                    └──────┬──────┘
       │                                                  │
       │ 1. TLS Handshake                                 │
       │    ClientHello + Client Certificate              │
       │─────────────────────────────────────────────────>│
       │                                                  │
       │ 2. Verify Client Certificate                     │
       │    - Signature valid                             │
       │    - Not expired                                 │
       │    - Not revoked (check CRL)                     │
       │    - CN matches node-id                          │
       │                                                  │
       │ 3. Server Certificate + ServerHello              │
       │<─────────────────────────────────────────────────│
       │                                                  │
       │ 4. Verify Server Certificate                     │
       │    - Signed by Control Plane CA                  │
       │    - CN matches expected hostname                │
       │                                                  │
       │ 5. Encrypted channel established                 │
       │    (mTLS successful)                             │
       │◄════════════════════════════════════════════════►│
       │                                                  │
```

## Implementation

### Certificate Manager Service

```typescript
interface CertificateManager {
  // CA Operations
  generateIntermediateCA(): Promise<CertificatePair>
  rotateIntermediateCA(): Promise<CertificatePair>
  
  // Node Certificate Operations
  signNodeCSR(csr: CSR, bootstrapToken: string): Promise<NodeCertificate>
  revokeNodeCertificate(nodeId: string, reason: RevocationReason): Promise<void>
  renewNodeCertificate(nodeId: string): Promise<NodeCertificate>
  
  // Validation
  verifyClientCertificate(cert: X509Certificate): Promise<VerificationResult>
  verifyServerCertificate(cert: X509Certificate): Promise<VerificationResult>
  isRevoked(serialNumber: string): Promise<boolean>
  
  // CRL Management
  generateCRL(): Promise<CRL>
  publishCRL(): Promise<void>
}
```

### mTLS Middleware

```typescript
// Fastify plugin for mTLS authentication
export const mtlsPlugin = fp(async (fastify: FastifyInstance) => {
  fastify.addHook('onRequest', async (request, reply) => {
    // Skip mTLS for health checks and registration
    if (request.url === '/health' || request.url === '/api/nodes/register') {
      return
    }

    // Get client certificate from TLS socket
    const socket = request.raw.socket as TLSSocket
    const clientCert = socket.getPeerCertificate()

    if (!clientCert || Object.keys(clientCert).length === 0) {
      return reply.status(401).send({ error: 'Client certificate required' })
    }

    // Verify certificate
    const result = await fastify.certificateManager.verifyClientCertificate(clientCert)
    
    if (!result.valid) {
      return reply.status(401).send({ 
        error: 'Invalid client certificate',
        reason: result.reason 
      })
    }

    // Extract node identity from certificate
    const nodeId = clientCert.subject.CN
    request.node = { id: nodeId, verified: true }
  })
})
```

### Edge Agent mTLS Configuration

```typescript
// Edge agent TLS configuration
const tlsConfig = {
  // Client certificate (node identity)
  cert: fs.readFileSync('/etc/edgecloud/node.crt'),
  key: fs.readFileSync('/etc/edgecloud/node.key'),
  
  // CA certificate to verify server
  ca: fs.readFileSync('/etc/edgecloud/ca.crt'),
  
  // Require server certificate
  rejectUnauthorized: true,
  
  // Server name verification
  servername: 'api.edgecloud.io',
  
  // Cipher suites (TLS 1.3 + secure 1.2)
  minVersion: 'TLSv1.2',
  maxVersion: 'TLSv1.3',
  ciphers: 'TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256:ECDHE-RSA-AES256-GCM-SHA384'
}

// HTTPS agent with mTLS
const agent = new https.Agent(tlsConfig)

// All requests use mTLS
const response = await fetch('https://api.edgecloud.io/tasks', {
  agent,
  headers: { 'Content-Type': 'application/json' }
})
```

## Certificate Rotation

### Automatic Rotation (Before Expiry)

```
Days before expiry:
  30d: Warning log, mark for rotation
  14d: Attempt automatic rotation
   7d: Alert admin if rotation failed
   1d: Emergency rotation attempt
   0d: Reject connections (expired)
```

### Rotation Process

1. Node detects certificate nearing expiry
2. Node generates new key pair + CSR
3. Node sends CSR to Control Plane (using current valid cert)
4. Control Plane verifies identity and issues new certificate
5. Node atomically replaces old certificate
6. Node confirms successful rotation
7. Control Plane revokes old certificate (after grace period)

## Revocation

### Revocation Reasons

- **Key Compromise**: Private key leaked or stolen
- **CA Compromise**: Intermediate CA compromised
- **Affiliation Changed**: Node no longer part of cluster
- **Superseded**: Replaced by new certificate
- **Cessation of Operation**: Node decommissioned

### CRL Distribution

```
CRL Endpoint: https://api.edgecloud.io/crl
Update Frequency: Every 6 hours
Max Age: 24 hours
Format: X.509 v2 CRL (RFC 5280)
```

## Security Considerations

1. **Private Key Protection**: Node private keys stored in TPM/HSM where available
2. **Certificate Binding**: Certificates bound to specific hardware (TPM attestation)
3. **Network Segmentation**: Control Plane CA only accessible from internal network
4. **Audit Logging**: All certificate operations logged to audit trail
5. **Rate Limiting**: CSR submissions rate-limited per IP/node

## Error Handling

| Error Code | Description | Resolution |
|------------|-------------|------------|
| `CERT_EXPIRED` | Node certificate expired | Automatic rotation or manual renewal |
| `CERT_REVOKED` | Certificate revoked by CA | Re-register node with new bootstrap token |
| `CERT_INVALID` | Certificate signature invalid | Check CA trust chain |
| `CERT_MISMATCH` | CN doesn't match node ID | Re-issue certificate |
| `TLS_HANDSHAKE_FAILED` | General TLS error | Check cipher suite compatibility |

## Testing mTLS

```bash
# Test with valid certificate
curl --cert node.crt --key node.key \
     --cacert ca.crt \
     https://api.edgecloud.io/health

# Test without certificate (should fail)
curl https://api.edgecloud.io/health
# Expected: 401 Client certificate required

# Test with expired certificate (should fail)
curl --cert expired.crt --key expired.key \
     --cacert ca.crt \
     https://api.edgecloud.io/health
# Expected: 401 Certificate expired
```
